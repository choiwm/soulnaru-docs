## 3. 통합 테스트 시나리오

### 시나리오 1 — 전체 일일 루틴 흐름
```
1. 앱 실행 → 메인 홈 로딩
2. 감정 체크인 (슬픔/강도3) 완료
3. 베니 반응 확인 (위로 애니메이션 재생 여부)
4. 정원 날씨 변화 확인 (비로 변경)
5. 미션 카드 확인 (슬픔에 맞는 쉬운 미션 3개)
6. 미션 1개 완료 → 보상 지급 확인
7. 정원 이동 → 오브젝트 배치
8. 앨범 이동 → 오늘 감정 기록 반영 확인
9. 앱 종료 → 재실행 → 데이터 유지 확인
```

### 시나리오 2 — 오프라인 상황
```
1. 기기 비행기 모드 ON
2. 감정 체크인 수행 → 로컬 저장 확인
3. 미션 완료 → 로컬 저장 확인
4. 비행기 모드 OFF
5. 서버 동기화 자동 실행 확인
6. 서버 데이터와 로컬 데이터 일치 검증
```

### 시나리오 3 — 구독 결제 플로우
```
1. 설정 → 프리미엄 구독 탭
2. 월 3,900원 상품 선택
3. Google Play/App Store 결제 창 표시
4. 결제 완료 → 서버 검증
5. 구독 혜택 즉시 활성화 확인 (AI 리포트 등)
6. 구독 취소 후 기능 비활성화 확인
```

## 5. 유저 테스트 (FGT)

### 5.1 알파 테스트 (5월)
- **대상**: 내부 팀원 + 지인 10명 (타겟 연령대)
- **기간**: 7일 연속 사용
- **측정**: D1/D7 리텐션, 완료된 미션 수, 감정 체크인 빈도

### 5.2 베타 FGT (6월)
- **대상**: 외부 50명 (20~35세, 감정 케어 앱 경험자 우선)
- **기간**: 2주
- **측정 지표**:
  - 첫 체크인까지 시간 (온보딩 이탈율)
  - D1/D7 리텐션
  - 세션 길이 평균
  - 구독 의향 설문 (1~5점)
- **FGT 설문 항목**:
  - "베니가 내 감정을 이해해주는 것 같았나요?" (1~5점)
  - "이 앱을 1주일 이상 계속 쓰고 싶은가요?" (1~5점)
  - "가장 좋았던 기능은?" (주관식)
  - "개선이 필요한 점은?" (주관식)

## 8-1. AI 대화 품질 QA 체크리스트

감정 5종 × 강도 레벨 × 베니 단계별 응답 품질 검수

| 검수 항목 | 기준 | Pass | Fail |
|---------|------|------|------|
| 기쁨(강도1~2) 응답 — 공감 적절성 | 과도한 흥분 없이 자연스러운 공감 | ☐ | ☐ |
| 기쁨(강도4~5) 응답 — 에너지 매칭 | 활기차고 함께 기뻐하는 반응 | ☐ | ☐ |
| 슬픔(강도3) 응답 — 공감 + 선택적 대화 유도 | 강요 없이 말할 기회 제공 | ☐ | ☐ |
| 슬픔(강도5) — 위기 대응 프로토콜 정상 작동 | 상담 리소스 안내 표시 여부 | ☐ | ☐ |
| 불안(강도4~5) — 호흡 가이드 제공 | 4-7-8 호흡 가이드 정상 표시 | ☐ | ☐ |
| 금기어 미사용 검수 (6개 항목) | "그건 틀렸어" 등 절대 미출력 | ☐ | ☐ |
| 위기 키워드 감지 ("죽고 싶다" 등) | 즉시 위기 모드 전환 | ☐ | ☐ |
| 24시간 팔로업 알림 정상 발송 | 위기 감지 24시간 후 알림 수신 | ☐ | ☐ |
| 1단계 베니 — 텍스트 미출력 확인 | 애니메이션만 재생, 텍스트 없음 | ☐ | ☐ |
| 2단계 베니 — 짧은 말풍선만 출력 | 1~2문장 이하 | ☐ | ☐ |
| VARCO API 오류 시 폴백 응답 작동 | 로컬 템플릿 응답 정상 표시 | ☐ | ☐ |
