## 2. 핵심 스크립트 명세

### 2.1 GameManager
```csharp
public class GameManager : BaseManager
{
    // 싱글톤
    public static GameManager Instance;
    
    // 주요 상태
    public GameState CurrentState { get; private set; }
    public UserProfile CurrentUser { get; private set; }
    public bool IsFirstLaunch { get; private set; }
    
    // 주요 메서드
    public void LoadScene(string sceneName, bool showLoading = true)
    public void SetGameState(GameState state)
    public void OnApplicationPause(bool pauseStatus)
    
    // 이벤트
    public event Action<GameState> OnGameStateChanged;
}

public enum GameState { Boot, Onboarding, MainHome, Garden, Album, Settings }
```

## 4. 로컬 vs 서버 저장 분류

| 데이터 | 로컬 (PlayerPrefs/SQLite) | 서버 (Firebase) |
|--------|--------------------------|----------------|
| 인증 토큰 | ✅ | ✅ |
| 유저 프로필 | 캐시 | ✅ 원본 |
| 감정 체크인 | 오프라인 큐 | ✅ 원본 |
| 베니 성장 상태 | 캐시 | ✅ 원본 |
| 정원 배치 | 캐시 | ✅ 원본 |
| 앱 설정 (알림 등) | ✅ | - |
| 구독 상태 | - | ✅ 원본 |

## 5. 데이터 동기화 전략

- **온라인**: 실시간 동기화 (체크인/미션 완료 즉시 서버 저장)
- **오프라인**: 로컬 큐 저장 → 연결 복구 시 자동 배치 업로드
- **충돌 해결**: 서버 타임스탬프 기준 (서버 우선 원칙)
- **동기화 주기**: 앱 포그라운드 진입 시 + 체크인/미션 이벤트 발생 시
