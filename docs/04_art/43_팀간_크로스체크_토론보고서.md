## 1. 아트/디자인팀 산출물 발표 — 토론 시작


  🎨
  아트/디자인팀 발표아티스트 A(UI) · B(2D) · C(스프라이트) · D(UI UI 프리팹) · E(파티클)



    🎨

      아티스트 A (UI 디자인)
      37번·38번 문서로 BreathingGuide UI 5종, Check-In UI 8종 PNG 시안을 완성했습니다. 390×844px 1x와 @2x 전부 생성했고 개발팀이 즉시 임포트 가능한 상태입니다.**
      한 가지 짚고 싶은 점**: 37번 문서에 Figma 인터랙티브 프로토타입이 여전히 ❌ 미완료로 표시되어 있습니다. Figma 툴 접근 문제로 실제 .fig 파일은 제공 못 했지만, PNG 스펙 + 인터랙션 수치가 문서에 전부 있으므로 개발팀이 문서 기준으로 구현 가능하다고 봅니다. 상태 업데이트가 필요합니다.




    🖌️

      아티스트 C (스프라이트)
      3~5단계 감정 스프라이트 75개를 256×256 RGBA PNG로 생성했습니다. 색상 팔레트는 11번 아트 스타일 가이드 기준입니다.**
      발견한 불일치**: 11번 문서 컬러 구성 섹션에 "3단계 액센트 컬러 = #FCA5A5(피치핑크)"로 명시되어 있는데, 39번 문서 헤더 색상 스와치는 #FCA5A5가 맞지만 5.2절 "컬러 구성" 표에는 3단계가 **민트 그린 계열**로 잘못 기재된 섹션이 있었습니다. 교정 필요합니다.




    🏗️

      아티스트 B (2D 이미지 제작)
      3~5단계 PNG 스프라이트 세트 실체 제작은 4/20 마감으로 진행 중입니다. 스펙 문서(39~41번)를 보니 해상도 단계 단계별 2D 해상도 수가 "512px / 256px / 128px"으로 명시됐는데, **12번 에셋 파이프라인 문서에는 해상도 단계 기준이 "512px / 256px"으로 다르게 되어 있습니다.** 어느 기준을 따라야 하나요?




    💻

      개발팀 리드
      모바일 최적화 관점에서 해상도 단계0:1000이 맞습니다. 12번이 구형 기준이고 39~41번이 최신 합의 결과입니다. **12번 에셋 파이프라인 문서를 즉시 수정해야 합니다.** 그리고 37번 BreathingGuide UI 구현 착수하려는데, 스펙 문서 들숨 멈춤(Hold-in) 상태의 **타이머 텍스트 색상이 37번에 명시되지 않았습니다**. 아티스트 A 확인 부탁드립니다.




    🎨

      아티스트 A (UI 디자인)
      Hold-in 상태 타이머 텍스트 색상은 **#5B21B6**(들숨 상태와 동일)으로 확정합니다. 37번 문서에 Hold-in 전용 색상 행이 누락되어 있었네요. 추가하겠습니다.




    📋

      기획팀
      38번 Check-In 감정 분류를 보니 감정이 **5종(기쁨·슬픔·화남·불안·평온)**인데, 06번 감정 데이터 분류 체계에는 "피곤(Tired)"이 별도 감정으로 추가되어 있고 13번 애니메이션 문서에도 `ANIM_Checkin_Tired`가 있습니다. **피곤 감정이 Check-In UI에서 빠진 이유가 있나요?**




    🎨

      아티스트 A (UI 디자인)
      스펙 작성 당시 GDD 01번 "5감정 시스템"을 기준으로 잡았습니다. 06번 문서와 13번 문서에 피곤이 별도로 있는 줄 몰랐습니다. **PM 판단이 필요한 이슈입니다.**




    🤖

      PM Alex (조정)
      **판정**: GDD 01번 기준 5감정이 공식 체계입니다. 피곤(Tired)은 "슬픔 강도 3+" 또는 별도 메모 입력으로 캡처 가능하며, 06번 문서의 피곤 항목은 내부 분류용 보조 태그입니다. 단, 13번 `ANIM_Checkin_Tired`는 "슬픔 체크인 대체 애니메이션"으로 재분류하고, 06번에 피곤=슬픔 강도4+ 매핑을 명시합니다. **이슈 CX-01 등록.**




    💻

      개발팀 리드
      09번 DB 스키마 이슈입니다. 29번 기술결정서에서 BennyDialogue 컬렉션을 Firestore에 별도 추가하기로 결정했는데, 09번 DB 스키마 문서에 이게 반영이 안 되어 있습니다(D-03 이슈). 09번을 업데이트해야 합니다.




    🧪

      QA팀 리드
      저희도 짚을 게 있습니다. 37번 BreathingGuide UI에 **접근성(Accessibility) 고려 사항이 전혀 없습니다**. 앱스토어 심사에서 Dynamic Type 미지원은 리젝 사유가 될 수 있어요. 또한 38번 CheckIn UI에서 **색약 사용자를 위한 대체 표현**(감정 이모지 외 텍스트 레이블 표시) 여부가 명시되지 않았습니다.




    🎨

      아티스트 A (UI 디자인)
      좋은 지적입니다. 37번과 38번에 접근성 섹션을 추가하겠습니다. 감정 카드에는 이미 이모지 + 텍스트 레이블이 같이 표시되도록 스펙에 있었는데, 문서에 명시적으로 기술하겠습니다. Dynamic Type은 최소 14pt에서 22pt까지 허용으로 스펙 추가합니다.




    📋

      기획팀
      39~41번 스프라이트 파일명 규칙이 `benny_s3_joy_01.png`인데, 36번 크로스체크 AR-04 이슈에서 `benny_stage3_joy_lv5.png` 포맷 통일 논의가 있었습니다. 현재 두 포맷이 혼재할 위험이 있어요. **어느 것이 공식인가요?**




    🤖

      PM Alex (조정)
      **판정**: `benny_s{stage}_{emotion}_{intensity:02d}.png` 형식으로 공식 확정. 이미 75개 파일이 이 규칙으로 생성·배포됐으므로 이 기준을 우선합니다. 36번 AR-04 이슈를 "해소됨"으로 마무리하고, 12번 에셋 파이프라인 네이밍 섹션에 이 규칙을 공식 반영합니다. **이슈 CX-02 등록.**




    💻

      개발팀 리드
      41번 5단계 나무의 **BennySeasonManager.cs 스크립트 스펙이 UI UI 프리팹 계층에만 있고, 09번 스크립트 명세서에는 없습니다.** 개발팀 입장에서 공식 스크립트 명세에 등록이 필요합니다.




    📋

      기획팀
      40번 4단계 열매 설명에서 "열매를 나눠주는 제스처(Benny_S4_Share 애니메이션)"가 언급됩니다. GDD 01번을 보면 4단계에서 **다른 유저와 열매를 교환하는 소셜 기능**이 기획되어 있습니다. 그런데 이 기능이 현재 Sprint 2에 포함됐는지 35번 문서에 명확하지 않습니다. Sprint 범위 명확화가 필요합니다.




    🤖

      PM Alex (조정)
      **판정**: 소셜 열매 교환 기능은 Sprint 5(5/1~5/7)로 연기 확정. 현재 4단계 에셋은 싱글 플레이 맥락으로만 사용. Benny_S4_Share 애니메이션은 Sprint 5 에셋 의존사항으로 등록. 40번 문서에 비고 추가. **이슈 CX-03 등록.**
