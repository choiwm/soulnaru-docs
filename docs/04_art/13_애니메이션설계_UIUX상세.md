## 1. 베니 애니메이션 클립 목록


  ✅ CX-01·CX-10 반영 (2026-04-16)****
  ① 피곤(Tired) 감정 재분류**: `ANIM_Checkin_Tired`는 **슬픔 강도4+ 대체 애니메이션**으로 재분류합니다.
  공식 감정 5종(기쁨·슬픔·화남·불안·평온) 외 피곤은 슬픔 강도 4~5 시 자동 선택되는 Variant입니다.**
  ② 스프라이트 기반 감정 애니메이션 연동 방식**:**
  감정 체크인 시 베니 표정 변화는 2D 스프라이트 전환** 방식으로 구현합니다.
  `benny_s{stage}_{emotion}_{intensity}.png` → `SpriteRenderer.sprite` 교체 → DOTween 0.3s fade.
  3D Animator는 신체 모션(꼬리 흔들기, 점프 등)만 담당하고, 표정은 스프라이트 레이어로 분리합니다.**
  ③ 구현 우선순위 (Sprint 2 → 3)**:

    - 🔴 Sprint 2 (4/14~4/20): `ANIM_Checkin_Joy`, `ANIM_Checkin_Sad`, `ANIM_Grow_Stage`
    - 🟡 Sprint 3 (4/21~4/27): 나머지 감정 체크인 클립 + `ANIM_Morning_Greeting`
    - 🟢 Sprint 4 이후: `ANIM_Dance`, `ANIM_Comfort`, 계절 변형 클립

## 2. 애니메이션 상태 머신 (Animator)


```
`
[Entry]
    ↓
[Idle_Sway] ← 기본 루프 상태
    ├── trigger: CheckinEmotion → [Checkin_[Emotion]] → Back to Idle
    ├── trigger: MissionComplete → [Mission_Complete] → Back to Idle
    ├── trigger: GrowStage → [Grow_Stage] → Back to Idle (높은 우선순위, 모든 상태에서 전환 가능)
    ├── trigger: TapBenny → [Tap_Reaction] → Back to Idle
    ├── bool: IsSleeping=true → [Sleep] ↔ Idle (시간 기반 자동 전환)
    └── trigger: DayGreeting → [Morning_Greeting] → Back to Idle

[Checkin_Joy] → 춤(Dance) 분기:
    - intensity >= 4: trigger: HighJoy → [Dance] → Back to Idle
    - intensity < 4: 직접 Idle 복귀

[Checkin_Sad/Tired] → 위로(Comfort) 분기:
    - intensity >= 4: trigger: DeepSad → [Comfort] → Back to Idle
`
```


### 트랜지션 설정


- **전환 시간**: 0.15s (빠른 감정 반응) ~ 0.3s (부드러운 상태 변화)
- **Has Exit Time**: 감정 반응 클립은 false (즉시 전환)
- **Interrupt Source**: Grow_Stage는 AnyState에서 전환 (최고 우선순위)

**UX 규칙**:

- 호흡 중 전화/알림 시: 자동 일시정지 + "언제든 다시 시작할 수 있어요" 안내

**기술 사항 (Unity)**:

- 씬: `BreathingGuide.unity` — Additive 로드 방식 (메인 위에 오버레이)
- 링 애니메이션: `DOTween` Scale 트윈 (Ease.InOutSine)
- 베니 연동: `BreathingManager.cs` → `BennyController.SetBreathingMode(true)`
- VARCO Sound: TTS 안내 음성 + 자연 환경음 AudioMixer 분리

## 8. 베니 성장 단계별 모델 교체 방식


### 8.1 교체 방식 결정: Animator Override Controller 채택


베니가 성장 단계(1→2→3→4→5단계)에 따라 3D 모델이 교체될 때, 애니메이션 상태 머신을 유지하면서 클립만 교환하는 **Animator Override Controller** 방식을 사용한다.

**선택 이유**:

- 동일한 Animator Controller 구조(상태 머신 로직) 유지 → 코드 변경 최소화
- 단계별 애니메이션 클립만 Override → 인터랙션 버그 방지
- Unity 공식 지원 표준 방식, 유지보수 용이


### 8.2 Animator Override Controller 구조


```
`
[Base Animator Controller]  ←  BennyBase.controller
  └── 상태 머신 공유 (Idle/Checkin/Dance/Grow 등 동일)

[Stage Override Controllers]
  ├── BennyStage1.overrideController  ← 새싹 전용 클립
  ├── BennyStage2.overrideController  ← 꽃봉오리 전용 클립
  ├── BennyStage3.overrideController  ← 개화 전용 클립
  ├── BennyStage4.overrideController  ← 열매 전용 클립
  └── BennyStage5.overrideController  ← 나무 전용 클립
`
```


### 8.3 단계별 교체 클립 목록


| 클립 슬롯 (Base) | Stage 1 (새싹) | Stage 2 (꽃봉오리) | Stage 3 (개화) | Stage 4 (열매) | Stage 5 (나무) |

| `ANIM_Idle_Sway` | Sprout_Idle | Bud_Idle | Bloom_Idle | Fruit_Idle | Tree_Idle |

| `ANIM_Checkin_Joy` | Sprout_Joy | Bud_Joy | Bloom_Joy | Fruit_Joy | Tree_Joy |

| `ANIM_Checkin_Sad` | Sprout_Sad | Bud_Sad | Bloom_Sad | Fruit_Sad | Tree_Sad |

| `ANIM_Grow_Stage` | (없음) | Sprout_Grow | Bud_Grow | Bloom_Grow | Fruit_Grow |

| `ANIM_Dance` | Sprout_Bounce | Bud_Dance | Bloom_Dance | Fruit_Dance | Tree_Dance |


### 8.4 런타임 교체 코드 (BennyController.cs)


```
`
// 성장 단계 달성 시 Animator Override Controller 교체
public void UpdateBennyStage(int newStage)
{
    // 교체 전 현재 상태 저장
    var currentState = _animator.GetCurrentAnimatorStateInfo(0);

    // Override Controller 교체
    AnimatorOverrideController overrideController = _stageControllers[newStage - 1];
    _animator.runtimeAnimatorController = overrideController;

    // 3D 모델 Prefab 교체
    Destroy(_currentBennyModel);
    _currentBennyModel = Instantiate(_bennyModelPrefabs[newStage - 1], transform);

    // 성장 애니메이션 트리거
    _animator.SetTrigger("GrowStage");
}

// Inspector에서 ScriptableObject로 단계별 리소스 참조
[SerializeField] private AnimatorOverrideController[] _stageControllers; // 5개
[SerializeField] private GameObject[] _bennyModelPrefabs;               // 5개
`
```


### 8.5 교체 시 주의사항


| 항목 | 처리 방법 |

| **교체 시점** | `ANIM_Grow_Stage` 클립 종료 후 교체 (End Event 활용) |

| **이전 상태 복원** | 교체 전 Idle 상태로 강제 전환 후 Override 적용 |

| **메모리 관리** | 이전 단계 모델 Prefab 즉시 `Destroy()` |

| **교체 중 탭 방지** | 교체 진행 중 `_isBennyTransitioning = true` 플래그로 입력 차단 |
