## 1. 에셋 제작 워크플로우

```
[기획] 오브젝트/사운드 리스트 확정 (기획팀)
    ↓
[디자인] 컨셉 스케치 + 스타일 가이드 적용 (아트팀)
    ↓
[제작] VARCO AI 생성 or 수동 제작 (아트팀 + AI팀)
    ↓
[검토] PM + 아트 디렉터 품질 검수
    ↓
[최적화] 폴리곤 감소, 텍스처 압축, 오디오 변환
    ↓
[Unity 임포트] 임포트 설정 적용
    ↓
[프리팹화] 프리팹 생성 + 네이밍 컨벤션 적용
    ↓
[Addressable 등록] 에셋 그룹 배정
    ↓
[테스트] 타겟 기기에서 성능 검증
    ↓
[릴리즈] 에셋 번들 빌드 + CDN 배포
```

## 2. 3D 에셋 제작 기준

| 항목 | 기준 | 비고 |
|------|------|------|
| **최대 폴리곤** | 캐릭터(베니): 3,000 / 오브젝트: 2,000 / 배경: 5,000 | |
| **텍스처 크기** | 캐릭터: 512×512 / 오브젝트: 256×256 ~ 512×512 | 2의 제곱 필수 |
| **텍스처 포맷** | Android: ETC2 / iOS: PVRTC or ASTC | |
| **UV 매핑** | UV 아일랜드 겹침 금지, 마진 2픽셀 이상 | |
| **좌표계** | Unity 기준 (Y-up, Z-forward) | |
| **스케일** | 1 Unity Unit = 1m | |
| **원점** | 오브젝트 하단 중심 | |

### 4.2 VARCO 3D 작업 플로우
```
[프롬프트 작성] 영문, 구체적 스타일 지정 포함
    ↓
[VARCO API 요청] format: glb, poly_limit 지정
    ↓
[결과물 검수] 품질/스타일 적합성 판단
    ├── 통과: 최적화 단계로
    └── 수정: 프롬프트 개선 후 재생성 (최대 3회)
    ↓
[Blender 최적화] 폴리곤 감소, UV 정리, 텍스처 베이크
    ↓
[Unity 임포트]
```

## 5. Unity 임포트 설정

### 5.1 텍스처 임포트 세팅
```
Texture Type: Sprite (2D) or Default (3D)
sRGB: On (컬러 텍스처) / Off (노멀맵)
Generate Mip Maps: Off (UI), On (3D 씬)
Max Size: 512 (오브젝트), 1024 (캐릭터), 2048 (배경)
Compression: Android=ETC2, iOS=ASTC 4x4
```

### 5.2 메시 임포트 세팅
```
Scale Factor: 1
Generate Lightmap UVs: On (정원 씬 정적 오브젝트)
Normals: Calculate (VARCO 생성물)
Blend Shape Normals: None
```

### 5.3 오디오 임포트 세팅
```
Load Type: 
  - BGM: Streaming (메모리 절약)
  - SFX 짧은 것(<2s): Decompress on Load
  - SFX 긴 것(>2s): Compressed In Memory
Compression: Vorbis (Android/iOS 공통)
Quality: BGM 70% / SFX 80%
Sample Rate: 44100 Hz
```

## 6. 에셋 최적화 가이드

| 항목 | 방법 |
|------|------|
| **Draw Call** | 동일 소재 오브젝트 Static Batching, GPU Instancing |
| **텍스처 메모리** | Sprite Atlas 활용, 불필요 Mip Map 제거 |
| **Overdraw** | 투명 오브젝트 최소화, 파티클 수 제한 (씬당 100개 이하) |
| **메모리** | Addressable로 씬 전환 시 미사용 에셋 해제 |
| **오디오** | BGM Streaming, SFX 중복 재생 제한 (최대 5개) |
