## 1. 에셋 제작 워크플로우




      🌸 정원 오브젝트 컨셉 — 단순 2D 형태 파스텔 스타일 (에셋 제작 타겟)



```
`[기획] 오브젝트/사운드 리스트 확정 (기획팀)
    ↓
[디자인] 컨셉 스케치 + 스타일 가이드 적용 (아트팀)
    ↓
[제작] VARCO AI 생성 or 수동 제작 (아트팀 + AI팀)
    ↓
[검토] PM + 아트 디렉터 품질 검수
    ↓
[최적화] 2D 해상도 감소, 이미지 레이어 압축, 오디오 변환
    ↓
[Unity 임포트] 임포트 설정 적용
    ↓
[프리팹화] 프리팹 생성 + 네이밍 컨벤션 적용
    ↓
[Addressable 등록] 에셋 그룹 배정
    ↓
[테스트] 타겟 기기에서 성능 검증
    ↓
[릴리즈] 에셋 번들 빌드 + CDN 배포`
```





## 2. 2D 에셋 제작 기준


      | 항목 | 기준 | 비고 |

      | 최대 2D 해상도** | 캐릭터(베니): 3,000 / 오브젝트: 2,000 / 배경: 5,000 |  |

      | **이미지 레이어 크기** | 캐릭터: 512×512 / 오브젝트: 256~512 | 2의 제곱 필수 |

      | **이미지 레이어 포맷** | Android: ETC2 / iOS: PVRTC or ASTC |  |

      | **UV 매핑** | UV 아일랜드 겹침 금지, 마진 2픽셀 이상 |  |

      | **좌표계** | Unity 기준 (Y-up, Z-forward) |  |

      | **스케일** | 1 Unity Unit = 1m |  |

      | **원점** | 오브젝트 하단 중심 |  |






## 3. 2D 에셋 제작 기준




      🎭 베니 감정 스프라이트 — 512×512px 기준 제작 예시



      | 항목 | 기준 |

      | **베니 스프라이트** | 512×512px 기준 (레티나 대응 2x) |

      | **UI 아이콘** | 128×128px (2x: 256×256) |

      | **배경 이미지** | 1080×1920px (9:16 기준) |

      | **포맷** | PNG (투명도 있는 경우) / JPG (배경) |

      | **압축** | Android: ETC2 RGBA / iOS: ASTC 4x4 |

      | **스프라이트 아틀라스** | 씬/기능별로 Atlas 그룹화 (최대 2048×2048) |

      | **피벗** | 기본 Center, 발 기준 캐릭터는 Bottom |







### 3-4. 에셋 파일 네이밍 규칙 (공식) ✅ CX-08 반영

아래 규칙은 39~41번 스펙 문서에서 확정된 공식 규칙입니다. 모든 에셋 파일에 적용합니다.

  | 에셋 종류 | 파일명 패턴 | 예시 |

  | 베니 감정 스프라이트 | `benny_s{n}_{emotion}_{intensity:02d}.png` | `benny_s3_joy_03.png` |

  | 베니 이미지 레이어 (BaseColor/Shadow/Highlight) | `Benny_S{n}_{Type}.png` | `Benny_S3_BaseColor.png` |

  | 계절 이미지 레이어 | `Tree_{Season}_BaseColor.png` | `Tree_Spring_BaseColor.png` |

  | UI 시안 | `{Scene}_{State}.png` | `BreathingGuide_idle.png` |

  | Prefab | `Benny_Stage{n}_{StageName}.prefab` | `Benny_Stage3_Bloom.prefab` |

  | 애니메이션 클립 | `ANIM_{ActionName}.anim` | `ANIM_Checkin_Joy.anim` |


**스프라이트 emotion 값**: `joy | sad | angry | anxious | calm`  |  **intensity**: 01~05

## 5. Unity 임포트 설정



### 5.1 이미지 레이어 임포트 세팅


```
`Texture Type: Sprite (2D) or Default (2D)
sRGB: On (컬러 이미지 레이어) / Off (그림자맵)
Generate Mip Maps: Off (UI), On (2D 씬)
Max Size: 512 (오브젝트), 1024 (캐릭터), 2048 (배경)
Compression: Android=ETC2, iOS=ASTC 4x4`
```



### 5.2 메시 임포트 세팅


```
`Scale Factor: 1
Generate Lightmap UVs: On (정원 씬 정적 오브젝트)
Shadows: Calculate (VARCO 생성물)
Blend Shape Shadows: None`
```



### 5.3 오디오 임포트 세팅


```
`Load Type:
  - BGM: Streaming (메모리 절약)
  - SFX 짧은 것(<2s): Decompress on Load
  - SFX 긴 것(>2s): Compressed In Memory
Compression: Vorbis (Android/iOS 공통)
Quality: BGM 70% / SFX 80%
Sample Rate: 44100 Hz`
```





## 6. 에셋 최적화 가이드


      | 항목 | 방법 |

      | Draw Call** | 동일 소재 오브젝트 Static Batching, GPU Instancing |

      | **이미지 레이어 메모리** | Sprite Atlas 활용, 불필요 Mip Map 제거 |

      | **Overdraw** | 투명 오브젝트 최소화, 파티클 수 제한 (씬당 100개 이하) |

      | **메모리** | Addressable로 씬 전환 시 미사용 에셋 해제 |

      | **오디오** | BGM Streaming, SFX 중복 재생 제한 (최대 5개) |
