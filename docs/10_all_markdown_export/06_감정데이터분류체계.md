## 1. 5감정 정의 및 측정 기준

**✅ CX-01 반영 (2026-04-16 팀간 크로스체크 결정)**

공식 감정 분류는 **5종 (기쁨·슬픔·화남·불안·평온)**으로 확정됩니다.

**피곤(Tired)**은 독립 감정이 아닌 내부 보조 태그로 재분류합니다:
슬픔 강도 4~5 체크인 시 자동으로 Tired 태그가 부여되며, AI 반응 선택에 활용됩니다.
Check-In UI(38번)에는 표시되지 않으며, `ANIM_Checkin_Tired`는 슬픔 강도4+ Variant 애니메이션입니다.

| 감정 | 영문 | 정의 | 강도 1 | 강도 3 | 강도 5 |
| --- | --- | --- | --- | --- | --- |
| 기쁨 | Joy | 긍정적 경험으로 인한 활기찬 상태 | 살짝 기분 좋음 | 명확한 기쁨 | 매우 행복, 들뜸 |
| 평온 | Calm | 안정적이고 균형 잡힌 상태 | 그럭저럭 괜찮음 | 차분하고 안정적 | 완전한 내적 평화 |
| 피곤 | Tired | 신체적/정신적 소진 상태 | 약간 지침 | 꽤 피곤함 | 극도의 소진, 번아웃 |
| 슬픔 | Sad | 상실감, 우울감, 무기력한 상태 | 살짝 우울 | 꽤 슬프고 무기력 | 극심한 슬픔, 울고 싶음 |
| 불안 | Anxious | 미래/상황에 대한 걱정과 긴장 | 약간 신경 쓰임 | 꽤 불안하고 긴장됨 | 극심한 불안, 패닉 |

## 2. 감정 입력 방식

### 2.1 UI 인터랙션 방법

- 메인 체크인 카드 : 5개 감정 아이콘 탭 선택

- 강도 슬라이더 : 1~5 수평 슬라이더 (기본값 3)

- 선택적 메모 : 텍스트 입력 (최대 100자, 키보드 수동 입력)

- 날씨 선택형 (대안 UX): 5가지 날씨 이미지로 감정 표현 (맑음/흐림/비/바람/눈)

- 적용 범위 결정 (2026-04-04 확정) : v1.0 기본 체크인의 대안 UX 로만 제공. 별도 "날씨로 기분 표현" 탭을 감정 체크인 팝업 하단에 토글로 추가.

- 매핑 규칙 : 맑음 → joy(3), 흐림 → calm(2)/tired(2), 비 → sad(3), 바람 → anxious(3), 눈 → calm(4)

- 강도 슬라이더 : 날씨 선택형에서도 강도 슬라이더는 유지 (기본값 3, 조정 가능)

- v1.0 제외 기능 : 날씨 선택형만으로 단독 체크인 완료 불가 — 반드시 5감정 아이콘 선택 후 날씨로 세부 표현 보완하는 방식으로 운영. v1.1 이후 단독 사용 가능 여부 재검토.

### 2.2 입력 데이터 형식

## 3. 감정 분류 알고리즘

### 3.1 기본 분류 (규칙 기반)

- 유저가 직접 선택한 감정을 Primary Emotion으로 저장

- 강도 값은 1~5 정수로 저장

### 3.2 메모 보조 분석 (AI 기반)

메모 입력 시 텍스트를 분석하여 Secondary Emotion Score 산출:

`Input: "월요일이라 너무 힘들다"
Output: {
"primary": "tired", // 유저 선택
"memo_sentiment": "negative",
"memo_keywords": ["힘들다", "월요일"],
"secondary_emotions": {
"tired": 0.7,
"anxious": 0.2,
"sad": 0.1
}
}` 

### 3.3 분류 우선순위

- 유저 직접 선택 (가중치 최고 — 자기보고 존중)

- 메모 텍스트 분석 (보조 참고용)

- 연속 패턴 분석 (7일 이상 데이터 축적 시 패턴 감지)

## 4. 감정 전환 매트릭스

| 이전 감정 → 현재 감정 | 기쁨 | 평온 | 피곤 | 슬픔 | 불안 |
| --- | --- | --- | --- | --- | --- |
| 기쁨 | 지속 | 안정화 | 주의 | 알림 | 알림 |
| 평온 | 긍정 | 지속 | 주의 | 주의 | 주의 |
| 피곤 | 회복 | 회복 | ⚠️ 누적피로 | 위험 증가 | 위험 증가 |
| 슬픔 | 회복 | 개선 | ⚠️ 주의 | ⚠️ 지속슬픔 | ⚠️ 복합감정 |
| 불안 | 회복 | 개선 | ⚠️ 주의 | ⚠️ 복합감정 | ⚠️ 지속불안 |

- ⚠️ 표시: 아래 조건 중 하나 해당 시 베니 특별 위로 모드 + AI 리포트 생성

- 단일 감정 지속 : 동일 감정 강도 3+ 가 3일 연속 기록

- 복합 부정 감정 : 피곤/슬픔/불안 3종 중 2종 이상이 강도 3+ 로 3일 내 교차 기록

- 고강도 지속 : 어떤 감정이든 강도 5가 2일 연속

## 6. 개인정보 보호 고려사항

### 6.1 감정 데이터 특수성

감정 데이터는 **민감정보**에 해당할 수 있으므로 아래 원칙을 준수한다.

### 6.2 익명화 처리

| 데이터 종류 | 처리 방법 |
| --- | --- |
| 메모 텍스트 | 서버 저장 전 개인식별정보 자동 마스킹 |
| 감정 패턴 분석 | 집계 데이터만 사용 (개별 메모 미사용) |
| AI 학습 데이터 | 완전 익명화 후 옵트인 동의 필요 |

### 6.3 데이터 보관 정책

| 데이터 | 보관 기간 | 삭제 방법 |
| --- | --- | --- |
| 감정 체크인 기록 | 회원 탈퇴 후 30일 | 자동 삭제 |
| 메모 텍스트 | 즉시 또는 최대 1년 (유저 설정) | 유저 직접 삭제 가능 |
| 집계 통계 | 익명화 후 최대 3년 | 연구 목적 보관 |

### 6.4 유저 권리

- 열람권 : 내 감정 데이터 전체 다운로드 (JSON 형식)

- 삭제권 : 특정 날짜 또는 전체 기록 삭제

- 동의 철회 : 언제든지 AI 분석 기능 비활성화 가능

- 이동권 : 데이터 포터빌리티 지원
