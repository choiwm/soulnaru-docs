## 2. 버전 넘버링 규칙

**Semantic Versioning 적용**: `MAJOR.MINOR.PATCH`

| 버전 타입 | 규칙 | 예시 | 상황 |
| --- | --- | --- | --- |
| MAJOR | 전략/방향 대규모 변경 | v1.0.0 → v2.0.0 | BM 전면 수정, 플랫폼 변경 |
| MINOR | 기능/섹션 추가 또는 수정 | v1.0.0 → v1.1.0 | 새 메카닉 추가, 챕터 추가 |
| PATCH | 오탈자, 소규모 내용 정정 | v1.0.0 → v1.0.1 | 수치 수정, 표현 개선 |

### 문서별 버전 헤더 (각 문서 최상단 적용)

`---
title: GDD 핵심구조
version: v1.0.0
status: draft | review | approved | deprecated
author: Alex (AI PM)
reviewer: wonman choi (PD)
last_updated: 2026-04-02
---` 

### 상태(Status) 정의

| 상태 | 의미 |
| --- | --- |
| draft | 초안 작성 중 |
| review | 리뷰 요청됨 |
| approved | 승인 완료, 팀 공유 가능 |
| deprecated | 구버전, 더 이상 유효하지 않음 |

## 3. Git 브랜치 전략 (문서 전용)

`main ← approved 문서만 (배포 가능 상태)
↑ merge via PR + 리뷰
develop ← 작업 중인 문서 통합
↑ merge via PR
docs/[파트명]/[작업] ← 각 문서 작업 브랜치
예) docs/planning/gdd-v1.1
docs/dev/api-spec-update
docs/biz/kpi-revision` 

### 커밋 메시지 규칙

`[문서파트] [동작]: [내용] (버전)

예시:
[GDD] feat: 베니 5단계 성장 조건 상세화 (v1.1.0)
[API] fix: 감정 체크인 응답 스키마 오류 수정 (v1.0.1)
[BM] update: 구독 가격 3,900원으로 확정 (v1.0.2)
[ALL] docs: 4월 1주차 리뷰 피드백 반영 (v1.1.0)`

## 5. 팀별 협업 워크플로우

### 5.1 문서 업데이트 프로세스

`[담당자]
↓ 1. 브랜치 생성: docs/[파트]/[작업명]
↓ 2. 문서 수정 + 버전 헤더 업데이트
↓ 3. CHANGELOG 하단에 변경 내용 추가
↓ 4. PR 생성 → PM(Alex) + PD(wonman) 리뷰 요청

[PM Alex]
↓ 5. 내용 검토 + 코멘트
↓ 6. 승인 시 develop 머지

[wonman PD]
↓ 7. 주요 변경 최종 승인
↓ 8. develop → main 머지 (approved 상태로 전환)` 

### 5.2 주간 문서 리뷰 사이클

| 요일 | 활동 |
| --- | --- |
| 월요일 | 이번 주 업데이트할 문서 목록 이슈로 등록 |
| 수~목요일 | 담당자 수정 완료 + PR 생성 |
| 금요일 | Alex 리뷰 + 머지 + 주간 변경 요약 Slack 공유 |

## 6. 외부 공유 전략

### 6.1 팀 내부 (개발팀, 기획팀, 아트팀)

- 도구 : GitHub (마크다운 직접 열람) 또는 Notion (GitHub 연동 미러링)

- 접근 : 레포 Collaborator 초대 또는 Notion 팀스페이스 공유

- 알림 : PR 머지 시 Slack #docs-update 채널 자동 알림

### 6.2 경영진 / 외부 심사 (경기게임제작지원)

- 도구 : PDF 변환 (마크다운 → PDF)

- 생성 방법 : pandoc 또는 VS Code Markdown PDF 확장

- 명명 : 소울나루_[문서명]_v[버전]_[날짜].pdf 
예: 소울나루_GDD_v1.0.0_20260402.pdf

- 배포 : 각 마일스톤(중간평가/최종평가)에 맞춰 패키징

### 6.3 실시간 협업 (비개발 팀원)

- 도구 : Notion 페이지 (GitHub 마크다운을 Notion으로 임포트)

- 코멘트 : Notion 인라인 댓글로 피드백 수집

- 반영 : 피드백 → GitHub PR으로 공식 반영

## 7. 도구 설치 및 실행 가이드

### PDF 변환 (단일 문서)

`# pandoc 사용
pandoc docs/01_planning/01_GDD_핵심구조.md \
-o output/소울나루_GDD_v1.0.0_20260402.pdf \
--pdf-engine=xelatex \
-V mainfont="Noto Sans KR" \
-V fontsize=11pt` 

### 전체 문서 PDF 일괄 변환 스크립트

`#!/bin/bash
# scripts/export_pdf.sh
VERSION="v1.0.0"
DATE=$(date +%Y%m%d)
OUTPUT_DIR="output/pdf_${VERSION}_${DATE}"
mkdir -p "$OUTPUT_DIR"

for f in docs/**/*.md; do
filename=$(basename "$f" .md)
pandoc "$f" -o "${OUTPUT_DIR}/소울나루_${filename}_${VERSION}_${DATE}.pdf" \
--pdf-engine=xelatex -V mainfont="Noto Sans KR" -V fontsize=11pt
echo "✅ $filename 변환 완료"
done
echo "🎉 전체 PDF 변환 완료: ${OUTPUT_DIR}"` 

### Notion 임포트

`Notion → 새 페이지 → ··· 메뉴 → Import → Markdown & CSV
→ .md 파일 선택 → 임포트 완료`
