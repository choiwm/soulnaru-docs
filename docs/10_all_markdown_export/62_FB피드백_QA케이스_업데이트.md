# 🧪 FB-13: QA 위기 미션 케이스 추가 (46번 업데이트)

✅ **FB-13 긴급 완료** — 위기 미션 Level 3 QA 케이스 3개 신규 추가. §1.4.1 검증 필수 (심사 기준).

## 1. 추가 배경

**46번 테스트계획서**에서 위기 미션 (강도 5) 케이스가 누락됨. 심사 시 §1.4.1 검증이 필수이므로 4/29 오전 완료 필요.

## 2. 신규 QA 케이스 (§1.4.1)

CIS-01 | 불안 강도 5 선택 → Level 3 위기 분기

불안 강도 5 체크인 시 즉시 위기 UI 노출

**시나리오:**

- CheckIn 씬 진입

- 감정: 불안(Anxiety) 선택

- 강도: 강도 5 버튼 탭

- 완료(Submit) 버튼 탭

**기대 결과:**

- VARCO 호출 없이 즉시 위기 UI 표시 (≤0.5s)

- 베니 말풍선: "지금 많이 힘든 것 같아요. 전문가 선생님과 이야기해 볼까요?"

- 1393 전문가 버튼 노출 (활성화 상태)

- Firestore 기록: Level3_Crisis 이벤트 저장

**기기:** Android (Galaxy S21), iOS | **마감:** 4/29 완료

CIS-02 | 불안 강도 5 연속 3회 → 누적 위기 판정

같은 감정 강도 5 연속 입력 시 위기 분기 다시 확인

**사전 조건:** CIS-01 테스트 통과 후 같은 기기/세션에서 진행

- 위기 UI에서 1393 버튼 탭 → 외부 앱 호출 또는 안내 페이지

- CheckIn 씬으로 복귀 (또는 새로운 CheckIn 세션 시작)

- 불안 강도 5 재입력 (2회차)

- 위기 UI 다시 노출 확인

- 4시간 후 불안 강도 5 재입력 (3회차)

- 위기 UI 다시 노출 확인 + FCM 팔로업 예약 여부 확인

**기대 결과:**

- 매 강도 5 입력마다 위기 UI 노출 (누적 횟수 추적)

- 3회차 후 Firestore 기록: 누적 횟수 3회 저장

- FCM 팔로업: 24h 뒤 알림 예약 확인 (Firebase 콘솔)

**기기:** 동일 기기 | **마감:** 4/29 완료

CIS-03 | 슬픔 강도 5 + 위기 키워드 입력 → Level 3 분기

슬픔 강도 5에서 "죽고 싶어" 등 위기 키워드 감지 시 즉시 위기 분기

**시나리오:**

- CheckIn 씬 진입

- 감정: 슬픔(Sadness) 선택

- 강도: 강도 4 또는 5 선택

- 추가 메모 입력 필드에 위기 키워드 입력: "죽고 싶어", "자살", "이렇게 살고 싶지 않아" 중 하나

- 완료(Submit) 버튼 탭

**기대 결과:**

- 강도에 관계없이 위기 키워드 감지 시 즉시 위기 UI 표시

- 베니 말풍선 + 1393 버튼 노출

- Firestore 기록: Crisis_Keyword 이벤트 + 검출된 키워드 저장

- 24h 팔로업 예약 (Level 3 동일)

**키워드 목록:** "죽고 싶어", "자살", "이렇게 살 수 없어", "끝내고 싶어" 등 (기획팀 03번 ACT 설계서 참조)

**기기:** Android, iOS | **마감:** 4/29 완료
