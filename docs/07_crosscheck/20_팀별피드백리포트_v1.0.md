**[DB스키마-09] 오프라인 로컬 큐 스키마 추가**

현재: DataSyncManager에서 오프라인 큐 존재 언급 → 실제 저장 방식 미정의

- **결정 및 명시 필요**: PlayerPrefs(단순) vs SQLite(구조적) vs 로컬 파일(JSON)
- 추천: `SQLite` (unity-sqlite-net 라이브러리) — 감정 데이터는 구조적 저장이 적합
- 스키마 예시:

`CREATE TABLE offline_queue (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  action_type TEXT NOT NULL,  -- "emotion_checkin" | "mission_complete"
  payload TEXT NOT NULL,       -- JSON 직렬화된 데이터
  created_at INTEGER NOT NULL, -- Unix timestamp
  synced INTEGER DEFAULT 0
);
`
