### ❹ 감정 메모 PII 마스킹 처리 — 18번엔 있는데 09번(DB스키마)에 구현 없음

**발견 팀**: AI/심리 ↔ 개발
**관련 문서**: 18번 개인정보처리방침, 09번 DB스키마
**문제**:

- 09번 DB스키마: `memo` 필드 정의는 있지만 **마스킹/암호화 처리 로직 없음**
- 개발팀이 실제 어떻게 구현해야 하는지 알 수 없음

**영향**: 감정 메모에 이름, 전화번호 등 개인정보 입력 시 그대로 서버 저장 → 개인정보 유출 위험
**수정 방향**:

```
`
09번 DB스키마에 메모 처리 섹션 추가:
1. 서버 수신 시 정규식으로 전화번호/이메일/주민번호 패턴 감지 → 마스킹
2. Firestore 저장 전 AES-256 암호화 (18번과 일치)
3. 클라이언트 → 서버 전송 시 TLS 1.3 강제

예시 코드:
def mask_pii(text: str) -> str:
    text = re.sub(r'\d{3}-\d{4}-\d{4}', '[전화번호]', text)
    text = re.sub(r'[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+', '[이메일]', text)
    return text
`
```

**발견 팀**: 개발 ↔ 비즈니스
**관련 문서**: 10번 API 명세, 17번 BM, 01번 GDD
**문제**:

**수정 방향**:

```
`

B. 씨앗 소액 충전 도입 → 결제 API에 씨앗 충전 엔드포인트 추가
   POST /payments/seeds/purchase { package_id: "seeds_100" }

```

### ❾ 타겟 연령 표기 불일치 — GDD vs 유저스토리맵

**발견 팀**: 기획 내부 교차
**관련 문서**: 01번 GDD, 02번 유저스토리맵

| 문서 | 타겟 연령 표기 |

| **01번 GDD** | `타겟 연령: 18~35세 (MZ세대 주)` |

```

`
```
