# 💾 대화 맥락(conversation_history) 저장 방식 기술 결정서

      Firestore 별도 컬렉션 vs 세션 메모리 — 최적 선택안
      보고: 2026-04-07 · 담당: 개발팀 · 마감: 2026-04-06 18:00



      ✅ 결정 사항: Firestore 별도 컬렉션 (BennyDialogue)**
      **장기 대화 맥락 보존 + 멀티턴 AI 대응 + 감정 분석 활용





## 🔍 문제 정의



### 현재 상황


        09번 DB 스키마:** "benny_response" 필드만 저장 표시 (단일 턴)
        - **다중 턴 구조:** 명세 없음
        - **VARCO API 호출:** 이전 대화 맥락 전송 방식 불명확



### 요구사항


        - 베니와의 연속적인 대화가 의미 있어야 함
        - "너는 어제 슬픔 강도 5였잖아. 요즘 어때?"라는 맥락 기반 대화 필요
        - 감정 트렌드 분석 (D1~D7, D30 회고)
        - VARCO API 호출 시 이전 대화 문맥 포함
        - 개발환경/테스트 환경에서도 유지되어야 함



        **💡 핵심 차이:****
        · 세션 메모리: 앱 재시작 시 모든 대화 소실
        · Firestore: 영구 보관 → 다음날 접속 시에도 참조 가능






## ⚖️ 대안 비교




          Option A

### 세션 메모리


            ✓ 장점**

              - 구현 간단 (Array in Memory)
              - 응답 속도 최빠름
              - API 호출 최소



            **✗ 단점**

              - 앱 재시작 시 모든 대화 소실
              - 다음날 접속 시 문맥 전혀 없음
              - 30일 회고 불가능
              - 감정 트렌드 분석 불가능
              - VARCO API 호출 불가능
              - 심리적 연속성 깨짐





          ✅ 선택안

### Firestore 별도 컬렉션


            **✓ 장점**

              - 모든 대화 영구 보관
              - 다음날 접속 시 맥락 유지
              - 감정 트렌드 분석 가능
              - 멀티턴 VARCO AI 대응
              - 30일 앨범과 통합
              - 심리적 연속성 보장
              - 보안: 유저별 데이터 격리



            **⚠️ 단점**

              - 구현 복잡도 ↑
              - Firestore 호출 비용 증가
              - 지연시간 +200~300ms
              - 데이터 양 증가 (보관비용)






### 정량 비교 분석



          | 평가 항목 | 세션 메모리 | Firestore 컬렉션 | 가중치 |



          | **구현 복잡도** | 5/5 ⭐⭐⭐⭐⭐ | 2/5 ⭐⭐ | 낮음 |

          | **응답 속도** | 5/5 ⭐⭐⭐⭐⭐ | 3/5 ⭐⭐⭐ | 중간 |

          | **심리적 연속성** | 1/5 ⭐ | 5/5 ⭐⭐⭐⭐⭐ | **매우 높음** |

          | **VARCO AI 대응** | 1/5 ⭐ | 5/5 ⭐⭐⭐⭐⭐ | **매우 높음** |

          | **감정 분석** | 1/5 ⭐ | 5/5 ⭐⭐⭐⭐⭐ | **매우 높음** |

          | **30일 회고** | 1/5 ⭐ | 5/5 ⭐⭐⭐⭐⭐ | **매우 높음** |

          | **비용 효율성** | 5/5 ⭐⭐⭐⭐⭐ | 2/5 ⭐⭐ | 낮음 |

          | **총점** | **19점 / 42점** | **38점 / 42점** | **승자** |





        **🎯 분석 결과:** Firestore 컬렉션 방식이**
        심리적 연속성, AI 대응, 감정 분석 측면에서
        압도적으로 우수 (38점 vs 19점)






## 📋 선택안: Firestore 별도 컬렉션 (BennyDialogue)



### 컬렉션 스키마


{
  "user_id": "string (FK, unique)",
  "created_at": "timestamp",
  "updated_at": "timestamp",
  "conversation_history": [
    {
      "turn": 1,
      "timestamp": "ISO 8601",
      "user_input": "오늘 정말 힘들었어",
      "user_emotion": "슬픔 강도 3",
      "benny_response": "그런 날도 있지. 베니가 함께할게.",
      "response_emotion_tag": "위로",
      "varco_context": {
        "emotion_history": ["기쁨 1", "평온 2", "슬픔 3"],
        "recent_missions": ["감정 일기", "음료 미션"],
        "user_mood_trend": "악화"
      }
    },
    {
      "turn": 2,
      "timestamp": "ISO 8601",
      "user_input": "어제는 정말 기했는데...",
      "user_emotion": "기쁨 강도 2",
      "benny_response": "어제는 정말 기쁜 날이었지! 그리고 오늘도...",
      "response_emotion_tag": "공감 + 연결",
      "varco_context": { ... }
    }
  ],
  "session_date": "2026-04-07",
  "daily_emotion_summary": {
    "감정들": [...],
    "dominant_emotion": "슬픔",
    "max_intensity": 4
  }
}



### 핵심 필드



          | 필드명 | 타입 | 설명 | VARCO 사용 |



          | user_input** | string | 사용자 메시지 | O (프롬프트에 포함) |

          | **user_emotion** | string | 감정 + 강도 (e.g. "슬픔 3") | O (감정 컨텍스트) |

          | **benny_response** | string | 베니 응답 | O (이전 응답 참고) |

          | **response_emotion_tag** | string | 응답 의도 (위로, 공감, 질문 등) | O (톤 조절) |

          | **varco_context** | object | VARCO에 전달할 문맥 | O (프롬프트) |

          | **session_date** | date | 세션 날짜 (조회 쿼리 최적화) | O (일자별 쿼리) |





### 조회 패턴



          | 사용 사례 | 쿼리 | 설명 |



          | **현재 세션 로드** | SELECT * FROM BennyDialogue WHERE user_id = 'xyz' AND session_date = TODAY ORDER BY turn DESC LIMIT 10 | 오늘 대화 최근 10턴 |

          | **VARCO 프롬프트** | SELECT conversation_history, varco_context FROM BennyDialogue WHERE user_id = 'xyz' AND session_date = TODAY | 현재 세션 전체 맥락 |

          | **30일 회고** | SELECT daily_emotion_summary FROM BennyDialogue WHERE user_id = 'xyz' AND session_date BETWEEN D1 AND D30 | 월간 감정 추이 |

          | **감정 트렌드** | SELECT dominant_emotion FROM BennyDialogue WHERE user_id = 'xyz' ORDER BY session_date | 일별 주요 감정 추적 |








## 🤖 VARCO AI API 연동



### 다중 턴 대화 플로우


```
[사용자 메시지 입력]
  ↓
[감정 체크인 시스템]
  ├─ emotion_text 분류
  ├─ emotion_intensity (1~5)
  └─ 저장: BennyDialogue.user_emotion
  ↓
[Firestore 쿼리]
  └─ 오늘 대화 히스토리 로드 (최근 10턴)
  ↓
[프롬프트 구성 - VARCO API]
  ├─ System: "You are Benny, an empathetic AI companion"
  ├─ Previous Context:
  │   ├─ Last 5 exchanges
  │   ├─ Today's emotion trend: [emotions]
  │   ├─ User's current mood: [emotion + intensity]
  │   └─ Recent missions completed: [list]
  └─ User Message: [current input]
  ↓
[VARCO 응답]
  └─ benny_response + emotion_tag
  ↓
[저장: BennyDialogue]
  ├─ turn: increment
  ├─ user_input
  ├─ benny_response
  ├─ timestamp
  └─ varco_context
  ↓
[UI 표시]
  └─ 베니 응답 + 이모션 이펙트
```



### VARCO 프롬프트 예시


You are Benny, a compassionate AI companion in the game "Soul Naru".

Context:
- User's current emotion: Sadness, intensity 3
- Today's emotion trend: Joy(1) → Anxiety(3) → Sadness(3)
- Recent completed missions: ["Emotion Journal", "Warm Beverage"]
- Previous exchange:
  User: "Today was really hard"
  Benny: "Those days happen. I'm here with you."

User input: "I just want to rest"

Respond with:
1. Empathetic message (50-100 chars)
2. Emotion tag (e.g., "reassurance", "understanding")
3. Next suggested action (optional mission)



### 성능 최적화


        - **캐싱:** 오늘 대화 히스토리 → 메모리 캐시 (앱 시작 시 로드)
        - **배치 업데이트:** 10분마다 Firestore에 동기화
        - **문맥 윈도우:** VARCO 프롬프트에 최근 10턴만 포함 (API 비용 절감)
        - **인덱싱:** (user_id, session_date, turn) 복합 인덱스
