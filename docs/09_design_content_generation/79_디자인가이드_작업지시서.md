# 디자인 가이드 및 디자인 작업지시서

> 목적: 디자이너가 소울나루 관련 디자인 문서를 한 번에 확인하고 즉시 정식 이미지 제작에 착수할 수 있도록 정리한다.  
> 핵심 변경: 기존 2D/PNG 스프라이트 제작 기준을 **2D 이미지·스프라이트 제작 기준**으로 전환한다.

## 1. 최우선 작업 원칙

- **2D 기준**: 모든 캐릭터/오브젝트/씬 이미지는 2D 이미지이 아니라 2D 일러스트, 2D 스프라이트, 2D UI 이미지로 제작한다.
- **Pretendard + 부드러운 톤**: 텍스트는 Pretendard Regular~Medium 중심. 과한 Bold, 과한 그림자, 과한 발광은 피한다.
- **앱 적용 우선**: 문서용 이미지보다 Unity UI에 바로 넣을 수 있는 해상도/투명 배경/파일명 규칙을 우선한다.
- **안전한 감정 UX**: 위기/불안/슬픔 화면은 자극적 표현을 금지하고, 부드럽고 차분한 시각 언어를 사용한다.

## 4. 산출물 규격

| 구분 | 권장 규격 | 파일 형식 | 제작 기준 |
|---|---|---|---|
| 캐릭터 스프라이트 | 512×512px / @2x 1024×1024px | PNG 투명 배경 | 정면 기준, 표정 5종, 단계별 일관성 유지 |
| 정원 오브젝트 | 512×512px | PNG 투명 배경 | 작게 보여도 형태가 읽히는 실루엣 |
| UI 화면 예시 | 1080×1920px | PNG | 모바일 Safe Area 고려 |
| 문서 삽입 예시 | 1080×720px | SVG/PNG | 문서 설명용, 실제 앱 적용 이미지는 별도 제작 |
| 아이콘 | 128×128px / 256×256px | PNG/SVG | 단색·2색 버전 동시 제공 권장 |

## 5. 2D 전환 지시사항

| 기존 표현 | 변경 기준 | 디자이너 작업 의미 |
|---|---|---|
| 2D / 2D 이미지 제작 | 2D 이미지 제작 | 평면 일러스트/스프라이트로 제작 |
| PNG 스프라이트 세트 | PNG 스프라이트 세트 | 단계별 정면/표정/상태 이미지 제작 |
| 단순 2D 형태 이미지 제작 | 단순한 2D 형태 언어 | 기하학적이고 부드러운 실루엣 사용 |
| 해상도 단계 | 해상도 단계 | 512px, 256px, 128px 축소 버전 제공 |
| 이미지 레이어/Shadow | 색상 레이어/그림자 레이어 | 베이스/하이라이트/그림자 레이어로 분리 |

## 9. 최종 체크리스트

- [ ] 2D/PNG 스프라이트가 아니라 2D PNG/SVG 기준으로 제작했는가?
- [ ] 투명 배경이 필요한 이미지는 alpha가 유지되는가?
- [ ] 모바일 Safe Area에서 텍스트가 잘리지 않는가?
- [ ] Pretendard Regular~Medium 기반으로 과한 Bold를 피했는가?
- [ ] 색상은 라벤더·화이트·연블루·연그린 중심으로 차분한가?
- [ ] 위기/불안 관련 화면이 자극적이지 않은가?
